# Archive Directory

This directory contains historical documents from the development of Braven Charts.

## Purpose

Files archived here are:
- ✅ **Historical reference material** - Implementation notes and status reports
- ✅ **No longer actively referenced** - Not part of current development workflow
- ✅ **Preserved for learning** - Document decisions, bugs fixed, and approaches taken

## Contents

### `/implementation-notes/`

Contains detailed implementation logs and status documents from feature development:

- **Axis Position Implementation** - Multiple documents tracking axis positioning system development
- **Animation Implementation** - Zoom and pan animation system notes
- **Bug Fixes** - Crosshair coordinates, line continuity, tooltip positioning
- **Optimization Notes** - Axis padding optimization, zoom/pan fixes
- **Status Reports** - Task completion reports, visual verification logs

These documents were created during active development to track progress, document decisions, and analyze bugs. They remain valuable for understanding:
- Why certain architectural decisions were made
- How specific bugs were diagnosed and fixed
- Evolution of feature implementations over time

### `PROJECT_STATUS.md`

Historical project status snapshot - superseded by current documentation in specs/ and task tracking.

## Organization Policy

**What gets archived:**
- Implementation notes from completed features
- Bug analysis documents after fixes are merged
- Status reports after milestones complete
- Historical project snapshots

**What stays in root:**
- Active planning documents (ARCHITECTURE_REFACTOR_PLAN.md)
- Current constitution updates (CONSTITUTION_UPDATE_v*.md)
- Core documentation (README.md, CONTRIBUTING.md, CHANGELOG.md)

**What goes to specs/:**
- Feature specifications
- Requirements documents
- Architecture decision records (ADRs)
- Technical design documents
