# Orchestra CLI

> Orchestrate your development workflow with structured sprints and AI-assisted task management.

[![npm version](https://badge.fury.io/js/orchestra-cli.svg)](https://www.npmjs.com/package/orchestra-cli)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

## What is Orchestra?

Orchestra is a development workflow tool that helps you:

- **Structure Work**: Break projects into sprints and tasks
- **Track Progress**: Monitor task completion across sprints  
- **Generate Documents**: Auto-create feature plans and task specs
- **Maintain History**: Git-integrated sprint completion tracking

## Quick Start

```bash
# Install globally
npm install -g orchestra-cli

# Initialize in your project
cd your-project
orchestra init

# Check status
orchestra status

# Prepare a sprint
orchestra prepare 001

# Verify completion
orchestra verify 001

# Complete the sprint
orchestra complete 001
```

## Commands

| Command | Description |
|---------|-------------|
| `orchestra init` | Initialize Orchestra in a project |
| `orchestra status` | Show current project status |
| `orchestra prepare <sprint>` | Prepare a sprint for work |
| `orchestra verify <sprint>` | Verify sprint completion criteria |
| `orchestra complete <sprint>` | Complete a sprint and archive |

## Getting Started Guide (`docs/getting-started.md`)

```markdown
# Getting Started with Orchestra

This guide walks you through setting up Orchestra in your project.

## Prerequisites

- Node.js 18 or higher
- A git-initialized project (recommended)

## Installation

### Global Installation (Recommended)

```bash
npm install -g orchestra-cli
```

### Local Installation

```bash
npm install --save-dev orchestra-cli
npx orchestra --help
```

## Initialize Your Project

Navigate to your project root and run:

```bash
orchestra init
```

This creates the `.orchestra/` directory with:

```
.orchestra/
├── manifest.yaml          # Sprint and task definitions
├── config.yaml            # Tool configuration
├── templates/             # Document templates
│   ├── feature-plan.md.hbs
│   └── task-spec.md.hbs
└── docs/                  # Generated documentation
```

## Define Your First Sprint

Edit `.orchestra/manifest.yaml`:

```yaml
version: "1.0"

project:
  name: my-project
  description: My awesome project

sprints:
  - id: "001"
    name: Foundation
    description: Set up project foundation
    status: not-started
    tasks:
      - id: "001.1"
        name: Project Structure
        description: Create initial project structure
        status: not-started
        
      - id: "001.2"
        name: Core Types
        description: Define core type definitions
        status: not-started
```

## Start Working

### 1. Prepare the Sprint

```bash
orchestra prepare 001
```

This will:
- Create sprint documentation in `.orchestra/docs/sprints/001/`
- Generate task spec files from templates
- Optionally create a git branch `sprint/001-foundation`

### 2. Work on Tasks

As you complete tasks, update their status in `manifest.yaml`:

```yaml
tasks:
  - id: "001.1"
    name: Project Structure
    status: completed  # Changed from not-started
```

### 3. Check Progress

```bash
orchestra status
```

Output:
```
📊 Orchestra Status: my-project

Current Sprint: 001 - Foundation
Progress: ████████░░ 50% (1/2 tasks)

Tasks:
  ✅ 001.1 Project Structure
  ⏳ 001.2 Core Types
```

### 4. Verify Completion

Before completing a sprint, verify all criteria are met:

```bash
orchestra verify 001
```

### 5. Complete the Sprint

```bash
orchestra complete 001
```

This will:
- Generate a completion report
- Create a git tag `sprint-001`
- Archive sprint documentation

## Next Steps

- Read the [Command Reference](commands.md) for detailed options
- See [Workflows](workflows.md) for common patterns
- Check [Configuration](configuration.md) for customization options

## orchestra init

Initialize Orchestra in a project directory.

### Usage

```bash
orchestra init [options]
```

### Options

| Option | Description | Default |
|--------|-------------|---------|
| `--name <name>` | Project name | Directory name |
| `--description <desc>` | Project description | Empty |
| `--no-templates` | Skip template generation | false |
| `--force` | Overwrite existing config | false |

### Examples

```bash
# Interactive initialization
orchestra init

# Non-interactive with options
orchestra init --name "my-project" --description "A cool project"

# Skip templates
orchestra init --no-templates
```

### Generated Files

```
.orchestra/
├── manifest.yaml
├── config.yaml
└── templates/
    ├── feature-plan.md.hbs
    └── task-spec.md.hbs

## orchestra prepare

Prepare a sprint for active development.

### Usage

```bash
orchestra prepare <sprint-id> [options]
```

### Arguments

| Argument | Description | Required |
|----------|-------------|----------|
| `sprint-id` | Sprint identifier (e.g., "001") | Yes |

### Options

| Option | Description | Default |
|--------|-------------|---------|
| `--branch` | Create git branch | true |
| `--no-branch` | Skip branch creation | - |
| `--branch-name <name>` | Custom branch name | Auto-generated |
| `--docs` | Generate documentation | true |
| `--no-docs` | Skip documentation | - |

### Examples

```bash
# Prepare sprint with defaults
orchestra prepare 001

# Custom branch name
orchestra prepare 001 --branch-name "feature/foundation-v2"

# Skip branch creation
orchestra prepare 001 --no-branch
```

### Generated Files

```
.orchestra/docs/sprints/001/
├── readme.md
└── tasks/
    ├── 001.1.md
    └── 001.2.md
```

## orchestra verify

Verify sprint completion criteria.

### Usage

```bash
orchestra verify <sprint-id> [options]
```

### Arguments

| Argument | Description | Required |
|----------|-------------|----------|
| `sprint-id` | Sprint identifier | Yes |

### Options

| Option | Description | Default |
|--------|-------------|---------|
| `--strict` | Fail on warnings | false |
| `--json` | Output as JSON | false |
| `--fix` | Auto-fix fixable issues | false |

### Examples

```bash
# Basic verification
orchestra verify 001

# Strict mode (warnings = failure)
orchestra verify 001 --strict

# JSON output
orchestra verify 001 --json
```

### Verification Checks

1. **Task Completion**: All tasks marked as completed
2. **Documentation**: Required docs exist
3. **Git Status**: Working directory clean (warning)
4. **Manifest Valid**: No schema errors

### Output Example

```
🔍 Verifying Sprint 001: Foundation

Checks:
  ✅ All tasks completed (5/5)
  ✅ Sprint documentation exists
  ✅ Task specifications complete
  ⚠️ Uncommitted changes detected (warning)
  ✅ Manifest schema valid

Result: PASSED (1 warning)
Sprint 001 is ready for completion.
```

## orchestra complete

Complete a sprint and archive.

### Usage

```bash
orchestra complete <sprint-id> [options]
```

### Arguments

| Argument | Description | Required |
|----------|-------------|----------|
| `sprint-id` | Sprint identifier | Yes |

### Options

| Option | Description | Default |
|--------|-------------|---------|
| `--tag` | Create git tag | true |
| `--no-tag` | Skip tag creation | - |
| `--tag-name <name>` | Custom tag name | `sprint-{id}` |
| `--report` | Generate completion report | true |
| `--no-report` | Skip report generation | - |
| `--force` | Complete even with warnings | false |

### Examples

```bash
# Complete with defaults
orchestra complete 001

# Custom tag
orchestra complete 001 --tag-name "v1.0.0-foundation"

# Force completion despite warnings
orchestra complete 001 --force
```

### Actions Performed

1. Run verification checks
2. Update manifest (set `status: completed`, add `completedAt`)
3. Generate completion report
4. Create git tag
5. Commit changes

### Generated Files

```
.orchestra/docs/sprints/001/
└── completion-report.md
```

# Configuration Reference

Orchestra uses two configuration files in the `.orchestra/` directory.

---

## manifest.yaml

The manifest defines your project structure, sprints, and tasks.

### Schema

```yaml
version: "1.0"  # Required: Schema version

project:
  name: string           # Required: Project name
  description: string    # Optional: Project description
  repository: string     # Optional: Git repository URL

sprints:
  - id: string           # Required: Unique sprint ID (e.g., "001")
    name: string         # Required: Sprint name
    description: string  # Optional: Sprint description
    status: string       # Required: not-started | in-progress | completed
    startedAt: string    # Optional: ISO date when started
    completedAt: string  # Optional: ISO date when completed
    
    tasks:
      - id: string       # Required: Unique task ID (e.g., "001.1")
        name: string     # Required: Task name
        description: string  # Optional: Task description
        status: string   # Required: not-started | in-progress | completed
        assignee: string # Optional: Task assignee
        dependencies: string[]  # Optional: List of task IDs
```

### Example

```yaml
version: "1.0"

project:
  name: braven-charts
  description: A Flutter charting library
  repository: https://github.com/user/braven-charts

sprints:
  - id: "001"
    name: Foundation
    description: Core infrastructure and types
    status: completed
    startedAt: "2024-01-01"
    completedAt: "2024-01-15"
    tasks:
      - id: "001.1"
        name: Core Types
        description: Define fundamental types and interfaces
        status: completed
      - id: "001.2"
        name: Base Classes
        description: Implement base chart classes
        status: completed
        dependencies: ["001.1"]

  - id: "002"
    name: Chart Types
    description: Implement specific chart types
    status: in-progress
    startedAt: "2024-01-16"
    tasks:
      - id: "002.1"
        name: Line Chart
        status: completed
      - id: "002.2"
        name: Bar Chart
        status: in-progress
      - id: "002.3"
        name: Pie Chart
        status: not-started
```

## config.yaml

Tool configuration and preferences.

### Schema

```yaml
# Git integration settings
git:
  enabled: boolean          # Enable git features (default: true)
  branchPrefix: string      # Branch prefix (default: "sprint/")
  tagPrefix: string         # Tag prefix (default: "sprint-")
  autoCommit: boolean       # Auto-commit on complete (default: true)

# Documentation settings
docs:
  outputDir: string         # Docs output directory (default: "docs")
  templateDir: string       # Template directory (default: "templates")
  generateOnPrepare: boolean  # Generate docs on prepare (default: true)

# Verification settings
verify:
  requireCleanWorkingDir: boolean  # Require clean git status (default: false)
  requireAllTasksComplete: boolean # Require all tasks done (default: true)
  customChecks: string[]    # Custom verification scripts

# Output settings
output:
  colors: boolean           # Enable colored output (default: true)
  timestamps: boolean       # Show timestamps (default: false)
  verbose: boolean          # Verbose by default (default: false)
```

### Example

```yaml
git:
  enabled: true
  branchPrefix: "feature/"
  tagPrefix: "v"
  autoCommit: true

docs:
  outputDir: "docs"
  templateDir: "templates"
  generateOnPrepare: true

verify:
  requireCleanWorkingDir: false
  requireAllTasksComplete: true
  customChecks:
    - "./scripts/lint-check.sh"
    - "./scripts/test-check.sh"

output:
  colors: true
  timestamps: false
  verbose: false
```

## Template Files

Templates use [Handlebars](https://handlebarsjs.com/) syntax.

### Available Variables

**Sprint Context:**
- `{{sprint.id}}` - Sprint ID
- `{{sprint.name}}` - Sprint name
- `{{sprint.description}}` - Sprint description
- `{{sprint.tasks}}` - Array of tasks

**Task Context:**
- `{{task.id}}` - Task ID
- `{{task.name}}` - Task name
- `{{task.description}}` - Task description

**Project Context:**
- `{{project.name}}` - Project name
- `{{project.description}}` - Project description

### Example Template

```handlebars
# {{sprint.name}}

Sprint ID: {{sprint.id}}
Status: {{sprint.status}}

## Description

{{sprint.description}}

## Tasks

{{#each sprint.tasks}}
### {{this.id}}: {{this.name}}

{{this.description}}

**Status:** {{this.status}}
{{#if this.dependencies}}
**Dependencies:** {{#each this.dependencies}}{{this}}{{#unless @last}}, {{/unless}}{{/each}}
{{/if}}

---
{{/each}}
```
```

## CI/CD Integration

### GitHub Actions Example

```yaml
name: Orchestra Verify

on:
  pull_request:
    paths:
      - '.orchestra/**'

jobs:
  verify:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - uses: actions/setup-node@v4
        with:
          node-version: '18'
      
      - run: npm install -g orchestra-cli
      
      - name: Verify Sprint
        run: |
          SPRINT=$(orchestra status --json | jq -r '.currentSprint.id')
          if [ "$SPRINT" != "null" ]; then

          fi

# Troubleshooting

Common issues and solutions.

---

## Installation Issues

### "orchestra: command not found"

**Cause:** Global npm bin directory not in PATH.

**Solution:**
```bash
# Find npm bin directory
npm bin -g

# Add to PATH (bash)
export PATH="$PATH:$(npm bin -g)"

# Or use npx
npx orchestra-cli status
```

### Permission Errors on npm install

**Solution:**
```bash
# Use a Node version manager (nvm)
nvm install 18
nvm use 18
npm install -g orchestra-cli
```

---

## Initialization Issues

### "Already initialized"

**Cause:** `.orchestra/` directory exists.

**Solution:**
```bash
# View existing config
cat .orchestra/manifest.yaml

# Or force reinitialize
orchestra init --force
```

### "Not a git repository"

**Cause:** Git integration enabled but no git repo.

**Solution:**
```bash
# Initialize git first
git init

# Or disable git integration
# Edit .orchestra/config.yaml
# git:
#   enabled: false
```

---

## Command Errors

### "Sprint not found: XXX"

**Cause:** Sprint ID doesn't exist in manifest.

**Solution:**
```bash
# Check available sprints
orchestra status --history

# Verify manifest syntax
cat .orchestra/manifest.yaml
```

### "Task has unmet dependencies"

**Cause:** Task depends on incomplete tasks.

**Solution:**
1. Check task dependencies in manifest
2. Complete dependent tasks first
3. Or remove the dependency

---

## Verification Failures

### "Uncommitted changes detected"

**Cause:** Git working directory has changes.

**Solution:**
```bash
# Commit or stash changes
git status
git add .
git commit -m "WIP"

# Or bypass (not recommended)
orchestra complete 001 --force
```

### "Task XXX not completed"

**Cause:** Task status not set to "completed".

**Solution:**
1. Edit `.orchestra/manifest.yaml`
2. Set task `status: completed`
3. Re-run verification

---

## Template Issues

### "Template not found"

**Cause:** Missing template file.

**Solution:**
```bash
# Regenerate templates
orchestra init --force

# Or create manually
mkdir -p .orchestra/templates
# Create template files
```

### "Template render error"

**Cause:** Invalid Handlebars syntax.

**Solution:**
1. Check template syntax
2. Verify variable names
3. Use `{{this.property}}` in loops

---

## Git Integration Issues

### "Failed to create branch"

**Cause:** Branch already exists or git error.

**Solution:**
```bash
# Check existing branches
git branch -a

# Delete existing branch
git branch -d sprint/001-name

# Or use custom name
orchestra prepare 001 --branch-name "feature/001-v2"
```

### "Failed to create tag"

**Cause:** Tag already exists.

**Solution:**
```bash
# Delete existing tag
git tag -d sprint-001

# Or use custom tag
orchestra complete 001 --tag-name "sprint-001-v2"
```

---

## Getting Help

If these solutions don't work:

1. Run with `--verbose` flag for more details
2. Check GitHub Issues
3. Open a new issue with:
   - Node.js version (`node --version`)
   - Orchestra version (`orchestra --version`)
   - Full error output
   - Steps to reproduce
