# Spec 004.1: BCP Refactor - Headless Agent Engine

**Status:** Draft
**Parent:** Spec 004 (BravenLab Studio Integration)

## 1. Objective
Transform `braven_chart_plus` (BCP) into a "Headless" SDK. It should provide the intelligence, state management, and rendering capability, but **zero** opinionated chat UI.

## 2. Refactoring Scope

### 2.1 UI Removal
**Delete/Move to Example:**
- `ChatInterface`
- `InlineChat`
- `MessageBubble`
- `ChartCard` (The container with buttons/shadows)
- `ConfigPanel` (The edit form)

**Retain in BCP:**
- `ChartWidget` (The pure canvas/rendering layer)
- `ChartConfiguration` (The model)
- `AgentService` (The logic)

### 2.2 The "Agent Engine" API
Instead of a Widget, BCP exposes a reactive controller:

```dart
class AgentController {
  // Streams for UI to consume
  Stream<List<Message>> get messages;
  Stream<AgentState> get state; // idle, thinking, tool_executing
  Stream<ChartConfiguration> get activeChart;
  
  // Actions for UI to call
  Future<void> sendUserMessage(String text);
  Future<void> updateChartProperty(String key, dynamic value);
  Future<void> cancelAction();
}
```

### 2.3 Tool Injection
BCP provides standard data tools. BLS injects app-specific tools.

```dart
// BCP defines:
abstract class AgentTool { ... }

// BCP provides:
class CalculateTool extends AgentTool { ... }
class RenderChartTool extends AgentTool { ... }

// BLS injects:
class FetchAthleteHistoryTool extends AgentTool { ... }
```

## 3. Deliverables
1.  Clean `lib/src/agentic/widgets` (only pure chart widgets remain).
2.  Enhanced `AgentService` with robust Stream API.
3.  Unit tests proving the agent can "chat" without any UI instantiated.
