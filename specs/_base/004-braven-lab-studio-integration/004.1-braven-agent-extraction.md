## 2. Core Domain Logic

### 2.1 Session State (`session_state.dart`)

MUST be immutable (Equatable).

```dart
enum ActivityStatus { idle, thinking, calling_tool, error }

class SessionState extends Equatable {
  final List<AgentMessage> history;
  final AgentMessage? pendingResponse;
  final ToolCall? activeTool;
  final ActivityStatus status;
  final ChartConfig? activeChart;

  // Implements copyWith...
}
```

### 2.2 Event Vocabulary (`agent_events.dart`)

Events emitted via `Stream<AgentEvent>`.

```dart
abstract class AgentEvent {}

class ChartCreatedEvent extends AgentEvent {
  final ChartConfig config;
  ChartCreatedEvent(this.config);
}

class ChartUpdatedEvent extends AgentEvent {
  final ChartConfig config;
  ChartUpdatedEvent(this.config);
}

class ErrorEvent extends AgentEvent {
  final String message;
  final Object? originalError;
}

class ThinkingEvent extends AgentEvent {
  final String description; // e.g. "Analyzing power curve..."
}
```

### 2.3 AgentSession Interface (`agent_session.dart`)

```dart
abstract class AgentSession {
  ValueListenable<SessionState> get state;
  Stream<AgentEvent> get events;

  /// Main entry point for user interaction
  Future<void> transform(String prompt, {List<BinaryContent>? attachments});

  /// Abort current operation
  Future<void> cancel();

  /// Cleanup
  void dispose();
}
```

## 3. The LLM Layer (The "Dumb Pipe")

### 3.1 Data Models (`agent_message.dart`)

```dart
enum MessageRole { user, assistant, system, tool }

class AgentMessage {
  final String id;
  final MessageRole role;
  final List<MessageContent> content;
  final Map<String, dynamic> metadata;
}

abstract class MessageContent {}

class TextContent extends MessageContent {
  final String text;
}

class ImageContent extends MessageContent {
  final String base64Data;
  final String mediaType;
}

class BinaryContent extends MessageContent {
  final List<int> data;
  final String mimeType;
  final String? filename;
}

class ToolUseContent extends MessageContent {
  final String id;
  final String toolName;
  final Map<String, dynamic> input;
}

class ToolResultContent extends MessageContent {
  final String toolUseId;
  final String output;
  final bool isError;
}
```

### 3.2 Provider Interface (`llm_provider.dart`)

```dart
abstract class LLMProvider {

  Future<LLMResponse> generateResponse({
    required String systemPrompt,
    required List<AgentMessage> history,
    List<AgentTool>? tools,
    LLMConfig? config,
  });
}
```

### 4.2 CreateChartTool (`create_chart_tool.dart`)

**Logic:**

1.  Takes JSON input.
2.  Validates via `ChartConfiguration.fromJson`.
3.  Returns success message.
4.  **Side Effect:** The `AgentSession` loop detects this tool was called successfully and captures the `ChartConfig` result to update `state.activeChart`.

## 6. Verification: The Smoke Test

**Goal:** Verify end-to-end "Synthetic Chart Generation" without breaking the build.

**Test Harness:** `example/lib/demos/headless_engine_demo.dart`

**Scenario:**

1.  Initialize `AgentSession` (with Mock or Real Anthropic Provider).
2.  Send Prompt: _"We are testing our now agent chart tool. Please use it to create a complex chart with as many properties as you can see and use. Create at least 4 chart series and add some variability in your data"_
3.  **Assert:**
    - `state.status` transitions `idle` -> `thinking` -> `idle`.
    - `events` stream emits `ChartCreatedEvent`.
    - `state.activeChart` is not null.
    - `state.activeChart.series.length` >= 4.
    - UI renders the chart widget using `state.activeChart`.

## 7. Implementation Rules

1.  **NO DOMAIN LOGIC:** `braven_agent` does NOT know what an "Athlete" or a "FIT File" is. It only knows `DataContext` and `BinaryContent`.
2.  **IMMUTABLE STATE:** Do not use `SetState` inside the Session. Emit new state objects.
3.  **ERROR SAFETY:** If LLM returns garbage JSON for a chart, catch it, return a `ToolResultContent` with `isError: true` so the LLM can self-corret. Do NOT crash the app.
