# Spec 004.2: BravenLab Studio UI/UX & Flow

**Status:** Draft
**Parent:** Spec 004 (BravenLab Studio Integration)

## 1. Objective
Define the user interface and interaction flow for BravenLab Studio (BLS), consuming the headless BCP engine.

## 2. Layout & Design System

### 2.1 The "Studio" Layout
Not a simple chat stream. A productivity workspace.

- **Left Panel:** Athlete/Session Context
    - Selected Athlete profile
    - File list (FIT files, CSVs)
    - Session history
- **Center Panel:** The Workbench
    - **Chat Mode:** Large chat interface for exploration.
    - **Dashboard Mode:** Grid of generated charts.
    - **Hybrid:** Split screen (Chat left, Active Chart right).
- **Right Panel:** Inspector (Contextual)
    - When Chart selected: Visual property editor (colors, axes).
    - When Data selected: Summary stats & metadata.

### 2.2 Chat Experience
- **Rich Bubbles:** BLS renders the message stream from BCP using its own design system.
- **Interactive Attachments:** "I've loaded your file." -> Clickable chip opening the Data Inspector.
- **Inline Editing:** Clicking a chart in the chat stream expands it to the Workbench.

### 2.3 Charts in Context
Charts are no longer disposable.
- **"Pin to Board":** User can take a chart from chat and pin it to a persistent dashboard.
- **Live Updates:** Modifying a chart in the Inspector sends updates back to the BCP `AgentController`, which might trigger the Agent to comment ("I see you changed the color to red").

## 3. Session Management
- **Persistence:** Chat history is saved to the local database (BLS responsibility).
- **Restoration:** When re-opening a session, BLS re-initializes the BCP `AgentController` with the historical messages.
